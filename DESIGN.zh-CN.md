# UI 设计规范

## 1. 设计定位

当前界面不是纯扁平白底站点，也不是高饱和霓虹风。它的核心气质是：

- 桌面优先的 AI 聊天工作台
- 默认深色，但完整支持浅色主题覆盖
- 整体近中性色，强调克制，而不是品牌色冲击
- 面板层次依靠暗面、轻边框、模糊和阴影建立
- 内容优先，装饰极少，交互反馈短促、安静、不喧闹

视觉关键词：

- `neutral`
- `dense but calm`
- `desktop workspace`
- `soft glass panel`
- `low-saturation grayscale`

## 2. 主题体系

当前界面采用双主题系统：

- 默认主题：深色
- 可选主题：浅色
- 未显式指定时，跟随系统 `prefers-color-scheme`

主题切换要求：

- 只切换 token，不改变布局结构
- 深浅主题的层级关系必须一致
- 浅色不是“反相黑白”，而是保留同样的面板逻辑、边框逻辑、圆角逻辑和阴影逻辑

## 3. 设计令牌

### 3.1 字体

- `--font-body`: `Inter`, `-apple-system`, `BlinkMacSystemFont`, `"Segoe UI"`, `Roboto`, `"Helvetica Neue"`, `"PingFang SC"`, `"Hiragino Sans GB"`, `"Microsoft YaHei"`, `Arial`, `sans-serif`
- `--font-display`: 与 `--font-body` 相同
- `--font-mono`: 当前实现中也与 `--font-body` 相同

说明：

- 当前规范统一使用 `Inter` 体系，不使用强个性的展示字体
- 代码和正文的区分更多依赖容器、边框、背景和字号，而不是强烈的字体反差

### 3.4 阴影与模糊

深色：

- `--shadow-xs`: `0 1px 3px rgba(0, 0, 0, 0.4)`
- `--shadow-sm`: `0 4px 16px rgba(0, 0, 0, 0.5)`
- `--shadow-md`: `0 12px 40px rgba(0, 0, 0, 0.6)`

浅色：

- `--shadow-xs`: `0 1px 2px rgba(0, 0, 0, 0.05)`
- `--shadow-sm`: `0 4px 12px rgba(0, 0, 0, 0.06)`
- `--shadow-md`: `0 12px 32px rgba(0, 0, 0, 0.08)`

模态层：

- `--modal-overlay`: 深色 `rgba(6, 8, 12, 0.62)`，浅色 `rgba(245, 247, 250, 0.82)`
- `--modal-shadow`: 使用大范围柔和阴影，不做强烈外发光

规则：

- 面板允许使用 `backdrop-filter: blur(...)`
- 允许阴影，但必须柔和、低对比、服务于层级，不得做霓虹发光

### 3.5 圆角

- `--radius-xl`: `20px`
- `--radius-lg`: `16px`
- `--radius-md`: `12px`
- `--radius-sm`: `8px`
- 胶囊 / 圆按钮：`999px`
- 图片圆角：`16px`

使用约定：

- 主壳层、主面板、模态：`20px`
- 输入框、编辑面板、次级容器：`16px`
- 工具按钮、状态块、代码块：`8px` 到 `12px`
- 圆形图标按钮：直径 `34px`，圆角 `50%` 或 `999px`

## 4. 排版规则

排版重点是清晰、密度适中、阅读稳定。

### 4.1 字号层级

已确认的主要字号：

- 空状态主标题：`28px / 700`
- Markdown `h1`: `22px / 600`
- Markdown `h2`: `18px / 600`
- Markdown `h3`: `16px / 600`
- 配置面板标题：`18px / 700`
- 主聊天正文：`14px`
- 品牌标题：`14.5px / 700`
- 普通辅助文字：`12px` 到 `13px`
- 标签 / 状态 / 工具按钮文字：`11px` 到 `12px`

### 4.2 文本风格

- 正文默认 `line-height` 在 `1.5` 到 `1.8`
- 次级文字一律靠 `--text-muted` / `--text-dim` 弱化，不通过过小字号硬压
- 标题依靠字重和间距建立层次，不依赖高饱和色
- 说明类文本尽量短，避免在面板中堆砌长段说明

## 5. 布局规范

### 5.1 应用壳层

- 最外层容器最大宽度：`1440px`
- 主聊天壳层最大宽度：`1260px`
- 顶层左右安全边距：`12px`
- 桌面垂直偏移：`3vh`
- 主视口高度：`94vh`

布局特征：

- 中央单主壳层，不做多列门户式首页布局
- 左侧为话题侧栏，右侧为聊天主区
- 整体是一个连续工作台，不是多个分离卡片的拼贴

### 5.2 侧栏与会话区

- 侧栏展开宽度：`264px`
- 侧栏折叠宽度：`68px`
- 聊天区占剩余空间
- 左右面板共享一套外轮廓，但中间拼接处去掉对应圆角

具体要求：

- `topics-panel` 仅保留左侧圆角
- `conversation-panel` 仅保留右侧圆角
- 左右之间不加明显缝隙

### 5.3 内容宽度

- 会话内容最大宽度：`900px`
- 输入区最大宽度：`900px`
- 常规 turn 最大宽度：跟随会话内容宽度
- 聊天左右 gutter：`24px`

说明：

- 正文不是铺满整块聊天区，而是居中收束，保证阅读稳定

## 6. 组件规范

### 6.1 主面板

主聊天面板和侧栏面板必须满足：

- 背景使用 `--bg-chat`
- `1px` 轻边框
- `20px` 外圆角
- 柔和中大型阴影
- 支持毛玻璃模糊

不要：

- 不要用高对比描边
- 不要用夸张悬浮阴影
- 不要在主面板表面叠加复杂纹理

### 6.2 顶部栏

顶部栏是轻量标题栏，不是营销导航栏。

规则：

- 最小高度 `60px`
- 横向布局
- 模型信息区居中
- 左右两侧放界面动作按钮
- 背景透明或近透明，不要做重色块压顶

模型选择按钮：

- 高度约 `28px`
- 胶囊圆角
- 默认透明
- hover 只做轻微底色变化
- caret 旋转用于开合反馈

### 6.3 侧栏

侧栏控制要小、轻、准。

规则：

- 顶部控制按钮直径 `34px`
- 默认透明或极轻背景
- 边框使用 `--border-strong`
- 文本 / 图标默认 `--text-muted`
- `:active` 可以轻微缩放到 `0.94`

侧栏行为：

- 支持折叠
- 折叠后仅保留核心控制
- 移动端使用遮罩层和抽屉逻辑

### 6.4 消息区

消息展示偏内容优先，不使用厚重气泡堆叠。

用户消息：

- 右对齐
- 气泡底色使用 `--bg-active`
- 圆角 `16px`
- 内边距 `10px 14px`
- 不使用明显描边和阴影

助手消息：

- 默认透明背景
- 不做厚卡片包裹
- 标题行显示模型 / 服务信息
- Footer 承载复制、重试、删除等操作

多模型对比：

- 桌面下允许双列显示
- `1200px` 以下折成单列

### 6.5 输入区

输入区是关键操作面板，必须保持稳定、克制、可扩展。

规则：

- 外层容器最大宽度 `900px`
- 面板圆角 `16px`
- 使用轻边框 + 内层伪元素底板形成轻微质感
- 不使用蓝色发光输入框
- 文本区最小高度 `52px`
- 最大高度 `200px`
- 输入文本 `14px`
- placeholder 使用弱化中性色

输入区底部工具栏：

- 按钮高度 `34px`
- 常规工具按钮圆角 `10px`
- 背景使用 `--bg-subtle`
- hover 切到 `--bg-hover`
- 图标按钮和文案按钮共用同一视觉体系

### 6.6 按钮

基础按钮：

- 默认字号 `13px`
- 字重 `600`
- 圆角 `10px`
- 内边距 `8px 16px`

主按钮 `btn-primary`：

- 背景 `--accent-strong`
- 文字使用深底色对比
- hover 可以略增强亮度和阴影

次按钮 `btn-secondary`：

- 背景 `--bg-subtle`
- `1px` 边框
- 默认文字 `--text-muted`

浮动圆按钮：

- 尺寸 `34px`
- 胶囊或正圆
- 允许使用 `blur(16px)`
- 用于关闭预览、图片操作、滚动到底部等悬浮动作

危险按钮：

- 仅在删除、清除、破坏性确认中使用
- 不得泛滥到普通流程按钮

### 6.7 标签与状态

状态块和轻标签要统一为低饱和体系。

规则：

- 状态块默认 `12px`
- 圆角 `12px`
- 左侧小圆点 `7px`
- 成功使用绿色
- 运行中使用橙色
- 错误使用红色
- 完成态使用中性浅灰

模型标签 / 芯片：

- 胶囊样式
- `1px` 轻边框
- 背景极淡
- 文字不能过亮抢主内容

### 6.8 Markdown / 内容渲染

聊天正文和预览内容要遵循统一内容样式。

标题：

- `h1`: `22px`
- `h2`: `18px`
- `h3`: `16px`
- `h1` / `h2` 带底部分隔线

段落和列表：

- 段落底部间距 `12px`
- 列表左缩进 `24px`

行内代码：

- 使用轻底色
- 内边距 `2px 6px`
- 圆角 `4px`

代码块：

- 背景使用 `--bg-active` 或 `--bg-subtle`
- `1px` 轻边框
- 容器圆角 `8px` 到 `10px`
- 工具条包含语言名和复制 / 预览按钮

引用：

- 左边线使用 `--accent`
- 文本使用 `--text-muted`
- 允许斜体

链接：

- 使用 `--accent-strong`
- hover 用下划线

表格：

- 横向可滚动
- 单元格边框使用轻边框
- 表头底色极淡

图片：

- 圆角 `16px`
- 不做厚边框相框

### 6.9 配置面板与模态

配置面板是完整的工作区，不是小弹窗。

规则：

- 外层遮罩带模糊
- `#configModal` 在桌面下使用接近整壳层的大面板
- 最大宽度跟随应用壳层，桌面 runtime 可收束到 `1160px`
- 面板圆角 `20px`
- `header / tabs / content / footer` 结构分明
- 内容区独立滚动

配置内部：

- tab 使用横向胶囊 / 轻按钮样式
- 状态条用胶囊 `status-pill`
- 表单控件在移动端取消显著 focus 发光
- 小屏下配置面板直接转全屏

确认弹窗：

- 使用更紧凑的 `420px` 左右宽度
- 结构简单
- 危险确认按钮允许高对比，但不要做强烈红色大面积铺底

## 7. 交互与动效

动效原则：短、轻、服务状态变化。

时长参考：

- 快速点击缩放：`0.12s` 到 `0.16s`
- hover / color / border 变化：`0.18s` 到 `0.2s`
- 输入面板、编辑面板状态切换：`0.24s`
- 侧栏展开收起：`420ms`

交互要求：

- hover 优先改变背景、边框、文字颜色
- active 允许非常轻的缩放
- focus 尽量用低强度 ring 或取消发光，避免破坏整体安静感
- 滚动条默认隐藏，交互时再显现
- shimmer 只用于生成中、连接中等状态，不得滥用

## 8. 响应式规则

已确认关键断点：

- `1200px` 以下：多助手列折叠为单列
- `901px` 以上：配置面板使用桌面网格布局
- `900px` 以下：配置面板改为全屏移动布局
- `1100px` 以下：聊天区上边距和空状态标题略收缩

移动端要求：

- Web 端容器铺满视口，不保留桌面外边距
- 主壳层圆角、阴影、模糊可取消，改为贴边布局
- 侧栏改为抽屉 + scrim
- 表单控件字号不低于 `16px`，避免移动端浏览器自动放大

## 9. Do / Don't

### Do

- 使用中性灰黑作为主视觉语言
- 保持桌面工作台式布局和稳定内容宽度
- 使用 `20 / 16 / 12 / 8 / pill` 这套圆角等级
- 让边框和阴影都保持低对比
- 让交互反馈尽量通过细微背景变化完成
- 为深浅主题同时考虑可读性和层级一致性
- 让消息区、输入区、配置区共享同一套 token

### Don’t

- 不要再使用当前仓库旧 `DESIGN.md` 那种纯白 Ollama 风格作为基准
- 不要引入高饱和品牌主色作为大面积背景
- 不要做夸张玻璃拟物、霓虹描边或彩色发光
- 不要把聊天区改成营销官网式 hero + feature block
- 不要让消息区出现厚重卡片墙
- 不要引入无必要的大字号标题和强装饰字体
- 不要让 hover、focus、active 反馈过猛
- 不要让浅色主题只是机械反色

## 10. 给实现 Agent 的提示

如果要继续实现这套界面风格，可直接使用这些提示：

- “构建一个桌面优先的 AI 聊天工作台，默认深色，使用中性灰黑配色和低对比玻璃面板，避免品牌色主导。”
- “主壳层最大宽度 1260px，左侧 264px 话题栏，右侧聊天区，两个面板拼接成一个整体。”
- “聊天正文最大宽度 900px，消息内容居中收束，用户消息右对齐气泡，助手消息保持透明文本流布局。”
- “输入区做成 16px 圆角的轻玻璃编辑面板，52px 起始高度，底部有 34px 高的工具按钮。”
- “所有操作按钮以中性色 hover 和轻微缩放为主，不使用强蓝色 focus 发光。”
- “配置面板在桌面上是大尺寸工作区弹层，在移动端切成全屏面板。”
